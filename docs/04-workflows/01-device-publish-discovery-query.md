# 设备发布、发现与查询流程

## 1. 记录原则

API 签名、参数、类职责和调用链以源码为准；本文只记录端到端验证中容易误判的发布发现约束。

## 2. 自动发布约束

除业务主动调用发布接口外，DM 会根据系统状态自动发布本机，使绑定/认证发起方可发现。

关键不变式：

- 熄屏一律停止发布。
- 亮屏且 Wi-Fi 或蓝牙至少一个开启才自动发布。
- WATCH / GLASSES 不自动发布。
- 自动化端到端 bind 前，source/sink 需在同一 Wi-Fi，且两端保持亮屏。

## 3. 排查提示

| 现象 | 优先排查 |
|---|---|
| source 发现不到 sink | 两端是否同 Wi-Fi、sink 是否亮屏、Wi-Fi/蓝牙是否开启 |
| 自动化测试偶发发现超时 | 板子是否熄屏；必要时先唤醒再跑 sink/source demo |
| 业务主动发布可见、自动发布不可见 | 检查屏幕状态和自动发布触发条件 |
